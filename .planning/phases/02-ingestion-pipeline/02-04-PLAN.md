  <action>
    Create src/prme/ingestion/pipeline.py implementing the full ingestion pipeline:

    1. IngestionPipeline class:
       - __init__(
           event_store: EventStore,
           graph_store: GraphStore,
           vector_index: VectorIndex,
           lexical_index: LexicalIndex,
           extraction_provider: ExtractionProvider,
           write_queue: WriteQueue,
         ):
         - Store all backend references
         - Create EntityMerger(graph_store)
         - Create SupersedenceDetector(graph_store)
         - _retry_tasks: dict[str, asyncio.Task] = {} for tracking retry tasks
         - _background_tasks: set[asyncio.Task] = set() for tracking extraction tasks

       - async ingest(
           content: str,
           *,
           user_id: str,
           role: str = "user",
           session_id: str | None = None,
           metadata: dict | None = None,
           wait_for_extraction: bool = False,
         ) -> str:
         - PHASE 1: Persist event immediately via write queue
           - Create Event model (content, user_id, role, session_id, metadata)
           - Submit event_store.append(event) through write_queue
           - Also submit lexical_index.index(node_id=str(event.id), content, user_id, "event") through write queue for content searchability (INGE-04)
           - Return event_id
         - PHASE 2: Extract and materialize (background or await)
           - Create asyncio.Task for _extract_and_materialize(event, event_id)
           - Add task to _background_tasks set with add_done_callback to discard
           - If wait_for_extraction: await the task
           - Return str(event.id)

       - async ingest_batch(
           messages: list[dict],
           *,
           user_id: str,
           session_id: str | None = None,
           wait_for_extraction: bool = False,
         ) -> list[str]:
         - Each dict in messages must have "content" and "role" keys, optional "metadata"
         - Process sequentially (ordered conversation history)
         - Call ingest() for each message with wait_for_extraction
         - Return list of event_ids

       - async _extract_and_materialize(event: Event, event_id: str) -> None:
         - try:
           - Call extraction_provider.extract(event.content, role=event.role)
           - Call validate_grounding(result, event.content)
           - Call _materialize(result, event, event_id)
         - except Exception:
           - Log error
           - Schedule retry: _schedule_retry(event, event_id)

       - async _materialize(result: ExtractionResult, event: Event, event_id: str) -> None:
         - For each extracted entity:
           - Call entity_merger.find_or_create_entity(name, entity_type, event.user_id, description=entity.description, session_id=event.session_id, evidence_event_id=event_id)
           - Store entity_id -> entity name mapping for relationship wiring
           - Index entity in vector store: write_queue.submit(lambda: vector_index.index(entity_id, entity.name + ": " + (entity.description or ""), event.user_id))

         - For each extracted fact:
           - Resolve temporal_ref if present: call _resolve_temporal(fact.temporal_ref) using dateparser
           - Determine node_type from fact.fact_type: "decision" -> NodeType.DECISION, "preference" -> NodeType.PREFERENCE, else -> NodeType.FACT
           - Create MemoryNode (node_type, content=f"{fact.subject} {fact.predicate} {fact.object}", user_id, session_id, scope=PERSONAL, lifecycle=TENTATIVE, confidence=fact.confidence, metadata={subject, predicate, object, temporal_ref, resolved_date}, evidence_refs=[event.id])
           - Submit node creation through write_queue
           - If subject entity was found/created above, create HAS_FACT edge from entity to fact node via write_queue
           - Call supersedence_detector.detect_and_supersede() for the new fact
           - Index fact in vector store and lexical index via write_queue

         - For each extracted relationship:
           - Look up source_entity and target_entity in the entity_id map
           - If both found, create edge (source -> target) with the relationship_type, via write_queue
           - If entity not found in map (not extracted or filtered by grounding), skip with a warning

         - If result.summary:
           - Index summary text in vector and lexical stores for searchability, linked to event_id

       - _resolve_temporal(temporal_ref: str | None) -> str | None:
         - If None, return None
         - Use dateparser.parse(temporal_ref, settings={"PREFER_DATES_FROM": "past", "RELATIVE_BASE": datetime.now(timezone.utc)})
         - If parsed successfully, return ISO format string
         - If not parseable, return None (keep original temporal_ref in metadata regardless)

       - _schedule_retry(event: Event, event_id: str, attempt: int = 1) -> None:
         - If attempt > 3: log error "Max retries exceeded for event {event_id}", return
         - Delay: 5 * (6 ** (attempt - 1)) seconds (5s, 30s, 180s per research)
         - Create asyncio.Task that sleeps for delay, then calls _extract_and_materialize again
         - Track in _retry_tasks[event_id]

       - async shutdown() -> None:
         - Cancel all background tasks in _background_tasks
         - Cancel all retry tasks in _retry_tasks
         - Wait for cancellations to complete

    2. Use structlog for logging throughout. Log at INFO for each ingestion (event_id, user_id), DEBUG for extraction details, WARNING for retry scheduling, ERROR for max retries exceeded.
