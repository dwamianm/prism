**DuckDB events table scope column with startup auto-migration, explicit-column SELECT queries, and multi-scope filtering on EventStore**

## Accomplishments
- Events table now includes `scope VARCHAR NOT NULL DEFAULT 'personal'` with index
- Startup auto-migration detects missing scope column on existing databases and adds it via ALTER TABLE, backfilling as 'personal'
- Graph nodes scope consistency verified at every startup
- EventStore persists Event.scope on INSERT and reads it back on SELECT with correct Scope enum mapping
- All `SELECT *` queries replaced with explicit column lists to avoid positional dependency issues
- Multi-scope query filtering added to `get_by_user()` and `get_by_hash()` with parameterized IN clause
- Full backward compatibility: callers without scope parameter continue to work identically

- `src/prme/storage/event_store.py` - Added scope to INSERT, replaced SELECT * with explicit column list, updated _row_to_event() column positions, added scopes parameter to get_by_user() and get_by_hash(), imported Scope

## Decisions Made
- DuckDB 1.4.x does not support `ALTER TABLE ADD COLUMN ... NOT NULL` -- used `DEFAULT 'personal'` without NOT NULL constraint for migration path. New databases created via CREATE TABLE still have the full NOT NULL DEFAULT constraint. This was discovered during Task 1 execution.
- Scope VARCHAR maps back to Scope enum via `Scope(value)` with a PERSONAL fallback for None values (safety net for migrated databases where ALTER TABLE adds a nullable column).

**1. [Rule 1 - Bug] DuckDB 1.4.x ALTER TABLE does not support NOT NULL constraint**
- **Found during:** Task 1 (schema migration)
- **Issue:** Plan specified `ALTER TABLE events ADD COLUMN scope VARCHAR NOT NULL DEFAULT 'personal'` but DuckDB 1.4.4 raises `ParserException: Adding columns with constraints not yet supported`
- **Fix:** Removed NOT NULL from ALTER TABLE migration; new databases still use CREATE TABLE with NOT NULL DEFAULT. Added code comment documenting the constraint limitation.
- **Files modified:** src/prme/storage/schema.py
- **Verification:** Migration test passes; existing rows backfilled as 'personal'
- **Committed in:** aaffde9 (Task 1 commit)
