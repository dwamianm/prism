<feature>
  <name>Epistemic Filtering and Composite Scoring</name>
  <files>src/prme/retrieval/filtering.py, src/prme/retrieval/scoring.py, tests/test_retrieval_scoring.py</files>
  <behavior>
    **Epistemic Filtering (Stage 4 — RFC-0005 §6, RFC-0003 §8):**
    - Input: list[RetrievalCandidate], RetrievalMode
    - Output: list[RetrievalCandidate] (filtered), list[ExcludedCandidate] (removed)
    - DEFAULT mode: Remove candidates where node.epistemic_type is in DEFAULT_EXCLUDED_EPISTEMIC (HYPOTHETICAL, DEPRECATED). Since MemoryNode does not yet have epistemic_type field, use a fallback: if node lacks epistemic_type attribute, treat as ASSERTED (included). This makes filtering forward-compatible.
    - EXPLICIT mode: Keep all candidates.
    - Each removed candidate produces an ExcludedCandidate with reason="epistemic_filtered:{type}".

    **Composite Scoring (Stage 5 — RFC-0005 §7):**
    - Input: list[RetrievalCandidate], ScoringWeights
    - Output: list[RetrievalCandidate] (sorted by score), list[ScoreTrace]

    Cases (compute_composite_score):
    - candidate with semantic_score=0.95, lexical_score=0.0, graph_proximity=0.0, salience=0.5, confidence=0.8, recency=30 days, epistemic=OBSERVED, path_count=1
      -> additive = 0.30*0.95 + 0.15*0.0 + 0.20*0.0 + 0.10*exp(-0.02*30) + 0.10*0.5 + 0.10*0.8
      -> = 0.285 + 0 + 0 + 0.10*0.5488 + 0.05 + 0.08 = 0.285 + 0.0549 + 0.05 + 0.08 = 0.4699
      -> * epistemic(OBSERVED=1.0) = 0.4699
      -> path_score = min(1/3, 1.0) = 0.333

    - Two candidates with identical composite_score but different IDs -> sorted by (-score, str(node.id)) for determinism

    - Per-request weight overrides: caller can pass ScoringWeights(w_semantic=0.5, w_lexical=0.1, w_graph=0.15, w_recency=0.1, w_salience=0.05, w_confidence=0.10) — weights must still sum to 1.0.
  </behavior>

    **GREEN phase:** Implement:

    **`src/prme/retrieval/filtering.py`:**
    - `def filter_epistemic(candidates: list[RetrievalCandidate], mode: RetrievalMode = RetrievalMode.DEFAULT) -> tuple[list[RetrievalCandidate], list[ExcludedCandidate]]`
    - If mode is EXPLICIT, return (candidates, [])
    - Otherwise, check each candidate's node for epistemic_type attribute (getattr with default ASSERTED). If epistemic_type in DEFAULT_EXCLUDED_EPISTEMIC, add to excluded list.

    **`src/prme/retrieval/scoring.py`:**
    - `def compute_composite_score(candidate: RetrievalCandidate, weights: ScoringWeights) -> ScoreTrace`
      - Calculate recency: `math.exp(-weights.recency_lambda * days_since_update)` where days_since_update = (now - node.updated_at).total_seconds() / 86400. If node has no updated_at, use created_at.
      - Look up epistemic weight from EPISTEMIC_WEIGHTS using getattr(node, 'epistemic_type', EpistemicType.ASSERTED) as key. Default 0.7 for unknown types.
      - Compute additive sum: w_semantic*semantic + w_lexical*lexical + w_graph*graph_proximity + w_recency*recency + w_salience*salience + w_confidence*confidence
      - Multiply by epistemic weight (multiplicative, not additive)
      - Round to 10 decimal places: `round(score, 10)`
      - Compute path_score: `min(candidate.path_count / 3.0, 1.0)`
      - Return ScoreTrace with all component values

    - `def score_and_rank(candidates: list[RetrievalCandidate], weights: ScoringWeights = DEFAULT_SCORING_WEIGHTS) -> tuple[list[RetrievalCandidate], list[ScoreTrace]]`
      - For each candidate, call compute_composite_score, set candidate.composite_score and candidate.score_trace
      - Sort by (-composite_score, -path_score, str(node.id)) for deterministic ordering
      - Return sorted candidates and their score traces
