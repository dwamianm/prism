    **Step 1: Add new enums to `src/prme/types.py`:**

    Add four new (str, Enum) classes following the existing pattern:

    - `EpistemicType`: OBSERVED, ASSERTED, INFERRED, HYPOTHETICAL, CONDITIONAL, DEPRECATED, UNVERIFIED (from RFC-0003 §3). Required for epistemic filtering (Stage 4) and epistemic weight lookup (Stage 5).
    - `QueryIntent`: SEMANTIC, FACTUAL, ENTITY_LOOKUP, TEMPORAL, RELATIONAL (used by query analysis to classify intent and select backends).
    - `RetrievalMode`: DEFAULT, EXPLICIT (from RFC-0003 §8 — DEFAULT excludes HYPOTHETICAL and DEPRECATED; EXPLICIT includes everything).
    - `RepresentationLevel`: REFERENCE, KEY_VALUE, STRUCTURED, PROSE, FULL (from RFC-0006 §4 — ordered by token cost, lowest to highest).

    Also add `EPISTEMIC_WEIGHTS` dict mapping EpistemicType to float weights for scoring:
    - OBSERVED: 1.0, ASSERTED: 0.9, INFERRED: 0.7, HYPOTHETICAL: 0.3, CONDITIONAL: 0.5, DEPRECATED: 0.1, UNVERIFIED: 0.5
    These are the epistemic multiplier values for the composite score formula (RFC-0005 §7). Mark the dict with a comment `# [HYPOTHESIS] — tunable per deployment`.

    Also add `DEFAULT_EXCLUDED_EPISTEMIC` set: {EpistemicType.HYPOTHETICAL, EpistemicType.DEPRECATED} — used by DEFAULT retrieval mode.

    **Step 2: Create `src/prme/retrieval/__init__.py`:**

    Create the retrieval package with imports from models.py and config.py. Export: RetrievalResponse, MemoryBundle, RetrievalCandidate, ScoreTrace, QueryAnalysis, ScoringWeights, PackingConfig.

    **Step 3: Create `src/prme/retrieval/models.py`:**

    All models use Pydantic BaseModel (or dataclass where frozen is needed). Follow existing codebase patterns.

    Models to create:

    - `QueryAnalysis`: Captures the output of query analysis (Stage 1).
      Fields: `query` (str), `intent` (QueryIntent), `entities` (list[str]), `temporal_signals` (list[dict] — each has type/value/resolved keys), `time_from` (datetime | None), `time_to` (datetime | None), `retrieval_mode` (RetrievalMode, default DEFAULT), `request_id` (UUID, default_factory=uuid4).

    - `CandidateSource`: Literal type or str enum for "GRAPH", "VECTOR", "LEXICAL", "PINNED".

    - `RetrievalCandidate`: Enriched candidate carrying all score components.
      Fields: `node` (MemoryNode), `paths` (list[str] — which backends produced it), `path_count` (int), `semantic_score` (float, default 0.0), `lexical_score` (float, default 0.0), `graph_proximity` (float, default 0.0 — 1-hop=1.0, 2-hop=0.7, 3-hop=0.4), `composite_score` (float, default 0.0), `score_trace` (ScoreTrace | None, default None), `representation` (RepresentationLevel | None, default None — set in packing stage), `token_cost` (int, default 0 — set in packing stage).

    - `ScoreTrace`: Explainable breakdown of the 8 composite score components per RFC-0005 §7.
      Fields (all float): `semantic_similarity`, `lexical_relevance`, `graph_proximity`, `recency_factor`, `salience`, `confidence`, `epistemic_weight`, `path_score`, `composite_score`.
      Use `model_config = ConfigDict(frozen=True)`.

    - `BundleSection`: Enum or Literal for "entity_snapshots", "stable_facts", "recent_decisions", "active_tasks", "provenance_refs".

    - `MemoryBundle`: The context-packed output per user decision.
      Fields: `sections` (dict[str, list[RetrievalCandidate]] — grouped by section type), `included_count` (int), `excluded_ids` (list[UUID] — IDs of candidates dropped for budget), `tokens_used` (int), `token_budget` (int), `budget_remaining` (int), `min_fidelity` (RepresentationLevel).

    - `RetrievalMetadata`: Metadata about the retrieval operation.
      Fields: `request_id` (UUID), `candidates_generated` (dict[str, int] — per-backend counts), `candidates_filtered` (int), `candidates_included` (int), `scoring_config_version` (str — from ScoringWeights.version_id), `timing_ms` (float), `backends_used` (list[str]), `embedding_mismatch` (bool, default False — flag per research recommendation).

    - `RetrievalResponse`: Top-level response per user decision.
      Fields: `bundle` (MemoryBundle), `results` (list[RetrievalCandidate] — the scored, ranked results before packing), `metadata` (RetrievalMetadata), `score_traces` (list[ScoreTrace] — always-on per research recommendation, one per result).

    - `ExcludedCandidate`: For full candidate audit.
      Fields: `node_id` (UUID), `reason` (str — e.g., "epistemic_filtered", "below_threshold", "budget_exceeded"), `composite_score` (float | None).

    **Step 1: Create `src/prme/retrieval/config.py`:**

    - `ScoringWeights(BaseModel)` with `model_config = ConfigDict(frozen=True)`:
      - `w_semantic: float = 0.30` — RFC-0005 default
      - `w_lexical: float = 0.15`
      - `w_graph: float = 0.20`
      - `w_recency: float = 0.10`
      - `w_salience: float = 0.10`
      - `w_confidence: float = 0.10`
      - `w_epistemic: float = 0.05` — multiplicative, not additive
      - `w_paths: float = 0.00` — tiebreaker only
      - `recency_lambda: float = 0.02` — decay rate for recency factor `exp(-lambda * days)`

      Properties:
      - `version_id` -> str: deterministic SHA-256 hash (first 12 chars) of all weight values concatenated. Use `hashlib.sha256()`. Format: `f"{self.w_semantic}:{self.w_lexical}:{self.w_graph}:{self.w_recency}:{self.w_salience}:{self.w_confidence}:{self.w_epistemic}:{self.w_paths}:{self.recency_lambda}"`.

      Validators:
      - `model_validator(mode='after')` named `validate_weights`: verify additive weights (semantic + lexical + graph + recency + salience + confidence) sum to 1.0 (within 1e-6 tolerance). Epistemic and paths are NOT included in the sum.

    - `PackingConfig(BaseModel)`:
      - `token_budget: int = 4096` — default context budget in tokens
      - `min_fidelity: RepresentationLevel = RepresentationLevel.REFERENCE` — minimum representation level
      - `overhead_tokens: int = 100` — reserved for JSON envelope/separators
      - `chars_per_token: float = 4.2` — character-based estimation default [HYPOTHESIS]
      - `graph_max_candidates: int = 50` — max candidates from graph traversal (RFC-0005 §4.1)
      - `vector_k: int = 50` — max candidates from vector search
      - `lexical_k: int = 50` — max candidates from lexical search
      - `graph_max_hops: int = 3` — max hops for graph neighborhood (1-3 per RFC)

    - `DEFAULT_SCORING_WEIGHTS = ScoringWeights()` — module-level constant for default config.
    - `DEFAULT_PACKING_CONFIG = PackingConfig()` — module-level constant for default packing.

    **Step 2: Add operations table to `src/prme/storage/schema.py`:**

    Add a new `CREATE TABLE IF NOT EXISTS operations` to the `initialize_database()` function, AFTER the existing tables.

    Schema (per research recommendation — minimal for RETRIEVAL_REQUEST, forward-compatible with Phase 5 TRST-08):
    ```sql
    CREATE TABLE IF NOT EXISTS operations (
        id VARCHAR PRIMARY KEY,
        op_type VARCHAR NOT NULL,
        target_id VARCHAR,
        payload JSON,
        actor_id VARCHAR,
        namespace_id VARCHAR,
        created_at TIMESTAMPTZ NOT NULL DEFAULT now()
    )
    ```

    Add an index: `CREATE INDEX IF NOT EXISTS idx_operations_op_type ON operations(op_type)`.
    Add an index: `CREATE INDEX IF NOT EXISTS idx_operations_created_at ON operations(created_at)`.

    This table stores RETRIEVAL_REQUEST records per RFC-0005 §9. It is separate from the event log per research recommendation (operational metadata, not conversation content).
