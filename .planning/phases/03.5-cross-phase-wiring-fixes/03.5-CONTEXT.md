## Implementation Decisions

### Integration Test Strategy
- Write integration tests that exercise each broken E2E flow end-to-end through the real pipeline
- Use real storage backends (DuckDB, Kuzu, tantivy) in temp dirs — these are integration wiring bugs, mocks won't catch them
- Name regression tests by E2E flow (test_e2e_scope_isolation, test_e2e_contradiction_detection, etc.) — semantic naming over audit-coupled naming
- Include a milestone-level assertion that all 29 v1.0 requirements are satisfied as a final gate

### Scope Backfill (GAP-03)
- Fix forward only — new writes include scope, existing data stays as-is. Full rebuild from event log is available when needed per portable artifact spec
- Scope stored as a tantivy field on each document, matching how Phase 03.2 already set up scope querying at read time
- Missing scope entries excluded from scope-filtered queries (strict) — prevents false positives, matches NSPC-05 scope isolation contract
- No rebuild command needed — wire scope into existing write paths only

### Config API Surface (GAP-01)
- Add epistemic_weights and unverified_confidence_threshold as individual __init__ params on RetrievalPipeline, matching existing pattern (scoring_weights, packing_config are already individual params)
- Fall back to module-level DEFAULT_* constants when params are None — backward compatible, existing callers that don't pass config still work
- Config read at startup only — matches pydantic-settings pattern, restart to apply env var changes
- No validation of unknown epistemic type keys — config is a dict, unknown keys silently ignored

### temporal_intent Forwarding (GAP-02)
- Add `temporal_intent=fact.temporal_intent` to `detect_and_supersede()` call in pipeline.py
- This is a one-line fix with high impact — enables the "assertion" branch that creates CONTRADICTS edges through automated ingestion

### CTXP Traceability
- Update existing Phase 3 VERIFICATION.md to add CTXP-01/02/03 entries under RETR-06 evidence
- Check off CTXP-01/02/03 boxes in REQUIREMENTS.md
- Keeps traceability accurate — the implementation lives in Phase 3's packing.py
