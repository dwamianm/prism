   - Add `epistemic_weights: dict[str, float] | None = None` after `packing_config` param
   - Add `unverified_confidence_threshold: float | None = None` after `epistemic_weights` param
   - Store as `self._epistemic_weights = epistemic_weights` (None means "use module default at call site")
   - Store as `self._unverified_confidence_threshold = unverified_confidence_threshold` (None means "use module default at call site")

2. **src/prme/retrieval/pipeline.py** -- In `retrieve()` method, pass stored values to Stage 4 and Stage 5:
   - Change the `filter_epistemic` call (currently `filter_epistemic(candidates, analysis.retrieval_mode)`) to:
     `filter_epistemic(candidates, analysis.retrieval_mode, unverified_threshold=self._unverified_confidence_threshold)`
   - Change the `score_and_rank` call (currently `score_and_rank(filtered, effective_weights)`) to:
     `score_and_rank(filtered, effective_weights, epistemic_weights=self._epistemic_weights)`

3. **src/prme/storage/engine.py** -- In `create()` method, add two kwargs to the `RetrievalPipeline(...)` constructor call (around line 184-191):
   - Add `epistemic_weights=config.epistemic_weights,` after the `packing_config=config.packing,` line
   - Add `unverified_confidence_threshold=config.unverified_confidence_threshold,` after that

NOTE: `filter_epistemic()` and `score_and_rank()` already accept these params with None defaults. No signature changes needed in filtering.py or scoring.py. The fix is only passing the values in from pipeline.py.

4. **src/prme/ingestion/pipeline.py** -- In `_materialize()`, find the `detect_and_supersede()` call (around line 400-407) and add `temporal_intent=fact.temporal_intent,` as a keyword argument after `evidence_event_id=event_id,`. This is a one-line addition.

**GAP-03: Lexical scope forwarding (4 changes)**

All four lexical write lambdas need a scope param added. Use Python default-argument capture to avoid late-binding closure bugs.

5. **src/prme/ingestion/pipeline.py** -- Phase 1 event lexical write (around line 151-155):
   - Current lambda: `lambda eid=str(event.id), c=content, uid=user_id: (self._lexical_index.index(eid, c, uid, "event"))`
   - Change to: `lambda eid=str(event.id), c=content, uid=user_id, sc=scope.value: (self._lexical_index.index(eid, c, uid, "event", sc))`

6. **src/prme/ingestion/pipeline.py** -- Fact lexical write (around lines 416-419):
   - Current lambda: `lambda fid=fact_node_id, fc=fact_content, uid=event.user_id, nt=node_type.value: (self._lexical_index.index(fid, fc, uid, nt))`
   - Change to: `lambda fid=fact_node_id, fc=fact_content, uid=event.user_id, nt=node_type.value, sc=fact_scope.value: (self._lexical_index.index(fid, fc, uid, nt, sc))`

7. **src/prme/ingestion/pipeline.py** -- Summary lexical write (around lines 462-464):
   - Current lambda: `lambda eid=event_id, s=result.summary, uid=event.user_id: (self._lexical_index.index(eid, s, uid, "summary"))`
   - Change to: `lambda eid=event_id, s=result.summary, uid=event.user_id, sc=scope.value: (self._lexical_index.index(eid, s, uid, "summary", sc))`
   - Use `scope.value` (the method-level `scope: Scope` param), NOT `fact_scope`, since summaries are event-level content.

8. **src/prme/storage/engine.py** -- `store()` lexical write (around lines 323-326):
   - Current lambda: `lambda nid=node_id, c=content, uid=user_id, nt=node_type.value: (self._lexical_index.index(nid, c, uid, nt))`
   - Change to: `lambda nid=node_id, c=content, uid=user_id, nt=node_type.value, sc=scope.value: (self._lexical_index.index(nid, c, uid, nt, sc))`

IMPORTANT: Use `scope.value` (not `scope`) since Scope is a (str, Enum) and LexicalIndex.index() expects `str | None`. Default-arg capture prevents late-binding closure bugs.
