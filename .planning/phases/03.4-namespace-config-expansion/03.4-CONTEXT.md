**Gathered:** 2026-03-01
**Status:** Ready for planning

<domain>
## Phase Boundary

Expand scope types from 3 to 6 per RFC-0004 and wire all [HYPOTHESIS]-marked parameters into PRMEConfig so they're tunable without code changes. Closes audit gaps NSPC-01 (3/6 namespace types) and TRST-07 (no systematic config mechanism).

</domain>

<decisions>
## Implementation Decisions

### Scope naming & compatibility
- Rename `ORG` to `ORGANISATION` (enum value "organisation") to match RFC-0004 exactly
- Clean break — no migration path, no aliases. Pre-release, no production data
- Add 3 new scopes: `AGENT`, `SYSTEM`, `SANDBOX` as enum values
- New scopes get docstring descriptions of intended semantics but no code behavior in this phase
- Update all ~16 files that reference `Scope.ORG` or `"org"`

### Config structure
- Nest `ScoringWeights` and `PackingConfig` under `PRMEConfig` as sub-fields
- Env vars follow pydantic-settings conventions: `PRME_SCORING__W_SEMANTIC=0.30`, `PRME_PACKING__TOKEN_BUDGET=4096`
- Config + defaults pattern: modules keep their `DEFAULT_*` constants as fallbacks; PRMEConfig overrides when provided
- Strict validation: ScoringWeights model_validator enforcing additive weights sum to 1.0 stays enforced at startup, even when loading from env vars

### Confidence matrix overrides
- Dict override with merge pattern: `confidence_overrides: dict[str, float]` on PRMEConfig
- Keys use "epistemic_type:source_type" format (e.g., "observed:user_stated")
- Overrides merge into the default matrix at startup — deployers only specify cells they want to change
- Enforce 0.0-1.0 range for all confidence override values; fail at startup if violated
- Allow new matrix entries not in the default set (forward-compatible for future epistemic/source types)

### Claude's Discretion
- SANDBOX/HARD_DELETE relationship encoding: docstring note vs constant — choose what's cleanest
- Epistemic weights override: dict pattern (like confidence matrix) or named fields (only 7 values) — choose what's most consistent
- TOML/YAML config file support: decide whether to add based on complexity vs value for an embeddable library
- Matrix override env vars: decide where to draw the line between env-configurable and programmatic-only
- `from_file()` convenience method: decide based on library ergonomics
- Config logging at DEBUG level: decide based on debuggability needs

</decisions>

- `PackingConfig` (retrieval/config.py): Already a Pydantic model with [HYPOTHESIS]-marked fields. Ready to nest under PRMEConfig.

### Established Patterns
- pydantic-settings with env_prefix and env_nested_delimiter for config loading
- Frozen Pydantic models for immutable config (ScoringWeights, ConfidenceMatrix)
- Module-level DEFAULT_* constants as singleton defaults (DEFAULT_SCORING_WEIGHTS, DEFAULT_PACKING_CONFIG, DEFAULT_CONFIDENCE_MATRIX)
- [HYPOTHESIS] comment markers on tunable parameters
