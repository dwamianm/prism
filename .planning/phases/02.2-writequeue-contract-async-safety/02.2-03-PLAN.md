  ```python
  async def _materialize(self, result, event, event_id, scope=Scope.PERSONAL):
      tracker = WriteTracker()
      # Create a new WriteQueueGraphWriter with this tracker for this event's writes
      from prme.ingestion.graph_writer import WriteQueueGraphWriter
      tracked_writer = WriteQueueGraphWriter(
          self._graph_store, self._write_queue, tracker=tracker
      )
      # Swap entity_merger and supersedence_detector to use tracked writer for this call
      entity_merger = EntityMerger(self._graph_store, tracked_writer)
      supersedence_detector = SupersedenceDetector(self._graph_store, tracked_writer)

      try:
          # ... existing materialization logic, but using entity_merger and
          # supersedence_detector local vars instead of self._entity_merger
          # and self._supersedence_detector ...
          # Also: for graph write_queue.submit calls in _materialize (fact nodes,
          # edges, relationship edges), record IDs in tracker:
          # After fact_node_id = await self._write_queue.submit(...create_node...):
          #     tracker.record_node(fact_node_id)
          # After edge creation via self._write_queue.submit(...create_edge...):
          #     tracker.record_edge(edge_id)
          # (entity_merger and supersedence_detector already use tracked_writer
          #  which records via the same tracker)
      except Exception as exc:
          logger.error(
              "ingestion.materialization_failed",
              event_id=event_id,
              exc_info=True,
          )
          # Rollback all graph artifacts from this event
          await tracker.rollback(self._graph_store, self._write_queue)
          logger.info(
              "ingestion.rollback_complete",
              event_id=event_id,
              rolled_back_nodes=len(tracker.node_ids),
              rolled_back_edges=len(tracker.edge_ids),
          )
          raise MaterializationError(
              f"Materialization failed for event {event_id}",
              event_id=event_id,
          ) from exc
  ```

  IMPORTANT: The direct `self._write_queue.submit(lambda fn=fact_node: self._graph_store.create_node(fn), ...)` calls for fact nodes and edges in `_materialize` should ALSO be tracked. After each such submit that creates a node, call `tracker.record_node(returned_id)`. After each such submit that creates an edge, call `tracker.record_edge(returned_id)`. This ensures rollback covers ALL graph artifacts, not just those from entity_merger/supersedence_detector.

  Note: Vector and lexical index writes within _materialize should NOT be rolled back (per research: "Log a warning for orphaned vector/lexical entries"). The tracker only covers graph artifacts.

- In `_extract_and_materialize()`, catch `MaterializationError` specifically and let it propagate to the retry mechanism (existing behavior via the outer `except Exception` block is fine since MaterializationError inherits from Exception).
