## Decisions Made
- Per-event WriteTracker created in _materialize() rather than a shared instance -- each event's graph artifacts are tracked and rolled back independently
- EntityMerger and SupersedenceDetector take both GraphStore (reads) and GraphWriter (writes) in __init__ -- dual-interface pattern provides structural write prevention while maintaining read access
- Pipeline.__init__ accepts optional graph_writer parameter with fallback to auto-creating WriteQueueGraphWriter -- backward-compatible API
- Concurrency tests use MockEmbeddingProvider (deterministic 8-dim vectors) and MockExtractionProvider (empty results) for CI speed
