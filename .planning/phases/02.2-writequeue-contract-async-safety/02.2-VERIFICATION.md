**One notable observation (not a blocker):**

The `supersede` lambda in `WriteQueueGraphWriter.supersede()` (line 153) captures `old_node_id`, `new_node_id`, and `evidence_id` by reference rather than by default-arg capture. The Plan specified the default-arg capture convention ("lambda n=node: ..."). Since these are function parameters (immutable within the call), not loop variables, there is no closure bug. The pattern is safe here — it is only a convention deviation, not a correctness issue.

| File | Line | Pattern | Severity | Impact |

| `src/prme/ingestion/graph_writer.py` | 153 | `supersede` lambda captures by reference not default-arg | Info | None — safe for function params |
