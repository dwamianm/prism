**src/prme/storage/embedding.py:**
Define the EmbeddingProvider Protocol and FastEmbed implementation.

```python
@runtime_checkable
class EmbeddingProvider(Protocol):
    @property
    def model_name(self) -> str: ...
    @property
    def model_version(self) -> str: ...
    @property
    def dimension(self) -> int: ...
    def embed(self, texts: list[str]) -> list[list[float]]: ...
```

`FastEmbedProvider` implementation:
- Constructor takes model_name (default "BAAI/bge-small-en-v1.5"), cache_dir (optional).
- Lazy-initialize fastembed.TextEmbedding on first embed() call (model download can be slow, don't block init).
- `embed(texts: list[str]) -> list[list[float]]`: Use TextEmbedding.embed() which returns a generator of numpy arrays. Convert to list[list[float]].
- Properties: model_name returns the configured model name, model_version returns the fastembed library version (or "fastembed-0.7.4"), dimension returns the configured dimension (384 for bge-small-en-v1.5).

**src/prme/storage/vector_index.py:**
Async VectorIndex wrapping USearch HNSW index with a DuckDB metadata table for UUID-to-key mapping and embedding model tracking.

```python
class VectorIndex:
    def __init__(self, conn: duckdb.DuckDBPyConnection, index_path: str, embedding_provider: EmbeddingProvider):
```

Constructor:
- Stores conn, index_path, embedding_provider.
- Creates vector_metadata table in DuckDB if not exists:
  ```sql
  CREATE TABLE IF NOT EXISTS vector_metadata (
      vector_key BIGINT PRIMARY KEY,
      node_id UUID NOT NULL,
      user_id VARCHAR NOT NULL,
      embedding_model VARCHAR NOT NULL,
      embedding_version VARCHAR NOT NULL,
      embedding_dim INTEGER NOT NULL,
      created_at TIMESTAMPTZ DEFAULT current_timestamp
  )
  CREATE SEQUENCE IF NOT EXISTS vector_key_seq START 1
  ```
  Index: idx_vector_node (node_id), idx_vector_user (user_id).
- Creates or loads USearch Index: `Index(ndim=embedding_provider.dimension, metric='cos', dtype='f32')`. If index_path file exists, load it. Otherwise create new.
- asyncio.Lock for writes.

Methods (all async):

1. **async index(self, node_id: str, content: str, user_id: str) -> int**: Generate embedding via embedding_provider.embed([content])[0]. Get next key from sequence (nextval('vector_key_seq')). Insert into vector_metadata table with model info. Add vector to USearch index. Save index to disk. Return the integer key.

2. **async search(self, query: str, user_id: str, *, k: int = 10) -> list[dict]**: Generate query embedding. Search USearch index. For each result, look up node_id from vector_metadata. Filter results by user_id (either via USearch filtered_search with a predicate that checks user_id in the metadata table, or post-filter -- prefer filtered_search if USearch supports a callable predicate, otherwise post-filter and request more results to compensate). Return list of dicts: {"node_id": str, "score": float, "distance": float}.

3. **async search_by_vector(self, vector: list[float], user_id: str, *, k: int = 10) -> list[dict]**: Same as search but takes pre-computed vector instead of text.

4. **async save(self)**: Persist USearch index to disk.

5. **async close(self)**: Save and clean up.

**User_id filtering strategy for USearch:**
USearch filtered_search accepts a predicate function `f(key: int) -> bool`. Build this by querying vector_metadata for all keys belonging to user_id, creating a set, and checking membership. For large vector counts, this is the bottleneck -- but it's sufficient for Phase 1. Cache the set per user_id if needed.

Alternative: Retrieve 3*k results from USearch without filtering, then post-filter by user_id via metadata lookup. Simpler but wastes computation. Use this approach if filtered_search has issues.

**Decision (Claude's discretion):** Single embedding model at a time per research recommendation. The vector_metadata table stores the model info per vector so the system knows if re-embedding is needed after a model switch.

Create the async LexicalIndex wrapping tantivy-py for BM25 full-text search.

```python
class LexicalIndex:
    def __init__(self, index_path: str):
```

Constructor:
- Build tantivy schema using SchemaBuilder:
  - `node_id`: text field, stored=True, tokenizer_name="raw" (exact match, not tokenized)
  - `content`: text field, stored=True, tokenizer_name="en_stem" (English stemmed tokenization for BM25)
  - `user_id`: text field, stored=True, tokenizer_name="raw" (exact match for filtering)
  - `node_type`: text field, stored=True, tokenizer_name="raw" (exact match for filtering)
- Create or open tantivy.Index with the schema at index_path.
- asyncio.Lock for writes.

Methods (all async):

1. **async index(self, node_id: str, content: str, user_id: str, node_type: str = "note") -> None**: Get IndexWriter (heap_size ~50MB). Add document with all four fields. Commit writer. Reload index so searcher sees new documents. Use asyncio.to_thread() for the blocking tantivy operations.

2. **async search(self, query_text: str, user_id: str, *, node_type: str | None = None, limit: int = 10) -> list[dict]**: Reload index. Create searcher. Build query string that combines content search with user_id filter. The query should search the content field for query_text AND filter to user_id.

   tantivy query syntax: Parse with the "content" field as default. Add a boolean clause for user_id filtering. Use `index.parse_query()` to create the query.

   Approach: Build query string as `{query_text}` and search content field, then post-filter by user_id. OR use tantivy's query parser: `f'content:({query_text}) AND user_id:"{user_id}"'` -- but tantivy-py's parse_query may not support this directly. Test both approaches.

   Simpler alternative: Use tantivy's BooleanQuery if available, or parse a combined query. If tantivy-py doesn't expose BooleanQuery, search content field and post-filter results by user_id field.

   Return list of dicts: {"node_id": str, "content": str, "score": float, "node_type": str}.

3. **async delete_by_node_id(self, node_id: str) -> None**: If tantivy-py supports term-based deletion, use it. Otherwise log a warning and skip (append-only model means deletion is rare). This is a stub for future re-indexing support.

4. **async close(self)**: Clean up writer if open.

**Key pitfall (from research):** Documents are NOT searchable until writer.commit() AND index.reload() are called. Always commit after adding and reload before searching. Wrap this in the implementation so callers never forget.

**Writer management:** tantivy-py's IndexWriter should be created per write operation (writer = index.writer(heap_size=50_000_000)), not held as a long-lived object. This avoids issues with concurrent write attempts. Alternatively, hold a single writer but protect with the asyncio.Lock.
