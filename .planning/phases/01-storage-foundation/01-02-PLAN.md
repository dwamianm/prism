<objective>
Build the DuckDB-based EventStore and GraphStore -- the two backends that live inside the same DuckDB database file.

Purpose: The event store is the immutable source of truth (STOR-01). The graph store enables relational queries over typed nodes and edges (STOR-02, STOR-03). Both share a DuckDB connection and must enforce user_id scoping (STOR-06). DuckPGQ is loaded as a community extension to enable SQL/PGQ pattern matching on graph data.

Output: Working EventStore (append/retrieve events) and GraphStore Protocol + DuckPGQ implementation (create/query nodes and edges with temporal validity and user scoping).
</objective>

   ```sql

       id UUID PRIMARY KEY,
       timestamp TIMESTAMPTZ NOT NULL,
       role VARCHAR NOT NULL,
       content TEXT NOT NULL,
   ```

   **nodes table:**
   ```sql
   CREATE TABLE IF NOT EXISTS nodes (
       id UUID PRIMARY KEY,
       node_type VARCHAR NOT NULL,
       user_id VARCHAR NOT NULL,
       session_id VARCHAR,
       scope VARCHAR NOT NULL DEFAULT 'personal',
       content TEXT NOT NULL,
       metadata JSON,
       confidence FLOAT DEFAULT 0.5,
       salience FLOAT DEFAULT 0.5,
       lifecycle_state VARCHAR NOT NULL DEFAULT 'tentative',
       valid_from TIMESTAMPTZ DEFAULT current_timestamp,
       valid_to TIMESTAMPTZ,
       superseded_by UUID,
       evidence_refs JSON,
       created_at TIMESTAMPTZ DEFAULT current_timestamp,
       updated_at TIMESTAMPTZ DEFAULT current_timestamp
   )
   ```
   Indexes: idx_nodes_user (user_id), idx_nodes_type (node_type), idx_nodes_lifecycle (lifecycle_state), idx_nodes_scope (scope).

   **edges table:**
   ```sql
   CREATE TABLE IF NOT EXISTS edges (
       id UUID PRIMARY KEY,
       source_id UUID NOT NULL REFERENCES nodes(id),
       target_id UUID NOT NULL REFERENCES nodes(id),
       edge_type VARCHAR NOT NULL,
       user_id VARCHAR NOT NULL,
       confidence FLOAT DEFAULT 0.5,
       valid_from TIMESTAMPTZ DEFAULT current_timestamp,
       valid_to TIMESTAMPTZ,
       provenance_event_id UUID,
       metadata JSON,
       created_at TIMESTAMPTZ DEFAULT current_timestamp
   )
   ```
   Indexes: idx_edges_source (source_id), idx_edges_target (target_id), idx_edges_type (edge_type), idx_edges_user (user_id).

**src/prme/storage/event_store.py:**
Create the async EventStore class wrapping DuckDB.

```python
class EventStore:
    def __init__(self, conn: duckdb.DuckDBPyConnection):
        self._conn = conn
        self._write_lock = asyncio.Lock()
```

Methods (all async, using asyncio.to_thread for sync DuckDB calls):

- `async append(self, event: Event) -> str`: Insert event into events table. Use parameterized query with event.id, event.timestamp, event.role, event.content, event.content_hash, event.user_id, event.session_id, event.metadata (json.dumps if not None). Return str(event.id). Use self._write_lock to serialize writes.

- `async get(self, event_id: str) -> Event | None`: SELECT * from events WHERE id = ?. Return Event model instance or None.

- `async get_by_user(self, user_id: str, *, session_id: str | None = None, limit: int = 100, offset: int = 0) -> list[Event]`: SELECT with WHERE user_id = ? and optional session_id filter. ORDER BY timestamp DESC. LIMIT/OFFSET.

- `async get_by_hash(self, content_hash: str, user_id: str) -> list[Event]`: Find events with same content hash for dedup detection.

Internal sync methods (_append_sync, _get_sync, etc.) do the actual DuckDB operations. Public async methods wrap them with asyncio.to_thread().

Important: The Event model has `frozen=True`, so when reconstructing from DB rows, use `Event.model_validate()` or construct with all fields provided.

**src/prme/storage/graph_store.py:**
Define the full-spec GraphStore Protocol (per user decision: NOT minimal CRUD, full-spec with all graph operations).

```python
@runtime_checkable
class GraphStore(Protocol):
    """Full-spec graph store interface for PRME."""

    # --- Node Operations ---
    async def create_node(self, node: MemoryNode) -> str: ...
    async def get_node(self, node_id: str, *, include_superseded: bool = False) -> MemoryNode | None: ...
    async def query_nodes(
        self,
        *,
        node_type: NodeType | None = None,
        user_id: str | None = None,
        scope: Scope | None = None,
        lifecycle_states: list[LifecycleState] | None = None,
        valid_at: datetime | None = None,
        min_confidence: float | None = None,
        limit: int = 100,
    ) -> list[MemoryNode]: ...

    # --- Edge Operations ---
    async def create_edge(self, edge: MemoryEdge) -> str: ...
    async def get_edges(
        self,
        *,
        source_id: str | None = None,
        target_id: str | None = None,
        edge_type: EdgeType | None = None,
        valid_at: datetime | None = None,
        min_confidence: float | None = None,
    ) -> list[MemoryEdge]: ...

    # --- Graph Traversal ---
    async def get_neighborhood(
        self,
        node_id: str,
        *,
        max_hops: int = 2,
        edge_types: list[EdgeType] | None = None,
        valid_at: datetime | None = None,
        min_confidence: float | None = None,
        include_superseded: bool = False,
    ) -> list[MemoryNode]: ...

    async def find_shortest_path(
        self,
        source_id: str,
        target_id: str,
        *,
        edge_types: list[EdgeType] | None = None,
    ) -> list[str] | None: ...

    # --- Supersedence ---
    async def get_supersedence_chain(
        self,
        node_id: str,
        *,
        direction: str = "forward",
    ) -> list[MemoryNode]: ...

    # --- Lifecycle Transitions ---
    async def promote(self, node_id: str) -> None: ...
    async def supersede(
        self,
        old_node_id: str,
        new_node_id: str,
        *,
        evidence_id: str | None = None,
    ) -> None: ...
    async def archive(self, node_id: str) -> None: ...
```

**src/prme/storage/duckpgq_graph.py:**
Implement DuckPGQGraphStore implementing the GraphStore Protocol. This plan covers node/edge CRUD and query_nodes/get_edges. Advanced traversal (neighborhood, paths, supersedence, lifecycle) is Plan 04.

```python
class DuckPGQGraphStore:
    def __init__(self, conn: duckdb.DuckDBPyConnection):
        self._conn = conn
        self._write_lock = asyncio.Lock()
```

Implement these methods in this plan (all async with asyncio.to_thread wrapping):

1. **create_node(node: MemoryNode) -> str**: INSERT into nodes table. Serialize evidence_refs as JSON array. Return str(node.id).

2. **get_node(node_id: str, *, include_superseded: bool = False) -> MemoryNode | None**: SELECT from nodes WHERE id = ?. If include_superseded is False, add WHERE lifecycle_state IN ('tentative', 'stable'). Reconstruct MemoryNode from row.

3. **query_nodes(...)**: Build dynamic WHERE clause from parameters. Default lifecycle_states to [TENTATIVE, STABLE] per user decision (filter to active by default). Apply temporal filter: if valid_at provided, add WHERE valid_from <= ? AND (valid_to IS NULL OR valid_to > ?). Apply min_confidence filter. ORDER BY created_at DESC. LIMIT.

4. **create_edge(edge: MemoryEdge) -> str**: INSERT into edges table. Return str(edge.id).

5. **get_edges(...)**: Build dynamic WHERE clause. Apply temporal and confidence filters. Return list of MemoryEdge.

Stub out the remaining Protocol methods (get_neighborhood, find_shortest_path, get_supersedence_chain, promote, supersede, archive) with `raise NotImplementedError("Implemented in Plan 04")`. This ensures the class structurally satisfies the Protocol for type checking while deferring complex logic.

Key implementation details:
- Use parameterized queries (?, not f-strings) for all user data to prevent SQL injection.
- DuckDB UUID type maps to Python UUID via str conversion. When inserting, cast to str. When reading, construct UUID from the returned value.
- evidence_refs stored as JSON array string in DuckDB. Serialize with json.dumps([str(x) for x in refs]), deserialize back to list[UUID].
- The _write_lock is shared with EventStore conceptually (both write to same DuckDB). For now each has its own lock; Plan 04's MemoryEngine will coordinate.
