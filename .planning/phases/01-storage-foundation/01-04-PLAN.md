**Lifecycle Transitions (per user decisions: full transition API in Phase 1):**

1. **promote(node_id: str) -> None**: Transition Tentative -> Stable.
   - Fetch current lifecycle_state from nodes table.
   - Validate transition using validate_transition() from prme.types.
   - If invalid, raise ValueError with clear message (e.g., "Cannot promote: node is {current_state}, only Tentative nodes can be promoted").
   - UPDATE nodes SET lifecycle_state = 'stable', updated_at = current_timestamp WHERE id = ?.

2. **supersede(old_node_id: str, new_node_id: str, *, evidence_id: str | None = None) -> None**:
   - Validate both nodes exist.
   - Validate old node can be superseded (current state must be Tentative or Stable per ALLOWED_TRANSITIONS).
   - UPDATE old node: lifecycle_state = 'superseded', superseded_by = new_node_id, updated_at = current_timestamp.
   - CREATE SUPERSEDES edge: new_node -> old_node (direction: the new node SUPERSEDES the old node). Edge has edge_type='SUPERSEDES', provenance_event_id=evidence_id, confidence=1.0.
   - Per user decision: evidence_id is optional for manual supersedence, required for automated (Phase 5). Don't enforce evidence_id requirement here -- just store it if provided.

3. **archive(node_id: str) -> None**: Transition any non-archived state -> Archived.
   - Validate transition.
   - UPDATE nodes SET lifecycle_state = 'archived', updated_at = current_timestamp WHERE id = ?.

**Graph Traversal (using DuckPGQ SQL/PGQ):**

4. **get_neighborhood(node_id: str, *, max_hops: int = 2, ...) -> list[MemoryNode]**:
   Try DuckPGQ first:
   ```sql
   FROM GRAPH_TABLE(memory_graph
       MATCH (a:nodes WHERE a.id = ?)-[e:edges]->{1,N}(b:nodes)
       COLUMNS (b.id, b.node_type, b.content, ...)
   )
   WHERE b.lifecycle_state IN ('tentative', 'stable')
   ```
   Apply optional filters: edge_types (WHERE e.edge_type IN ...), valid_at (temporal filter on b), min_confidence (WHERE b.confidence >= ?).
   If include_superseded is False (default), filter to active lifecycle states.

   **DuckPGQ fallback:** If the GRAPH_TABLE query fails (segfault, unsupported pattern), fall back to a recursive CTE on the edges table:
   ```sql
   WITH RECURSIVE neighborhood AS (
       SELECT target_id AS id, 1 AS depth FROM edges WHERE source_id = ?
       UNION
       SELECT e.target_id, n.depth + 1 FROM edges e
       JOIN neighborhood n ON e.source_id = n.id
       WHERE n.depth < ?
   )
   SELECT DISTINCT nodes.* FROM neighborhood JOIN nodes ON neighborhood.id = nodes.id
   WHERE nodes.lifecycle_state IN ('tentative', 'stable')
   ```
   Wrap the DuckPGQ attempt in try/except. Log a warning if falling back.

5. **find_shortest_path(source_id: str, target_id: str, ...) -> list[str] | None**:
   Try DuckPGQ:
   ```sql
   FROM GRAPH_TABLE(memory_graph
       MATCH p = ANY SHORTEST (a:nodes WHERE a.id = ?)-[e:edges]->+(b:nodes WHERE b.id = ?)
       COLUMNS (vertices(p) AS path_vertices)
   )
   ```
   Extract node IDs from path vertices. Return list of UUID strings or None if no path.

   Fallback: BFS via recursive CTE on edges table.

6. **get_supersedence_chain(node_id: str, *, direction: str = "forward") -> list[MemoryNode]**:
   Direction "forward" = what replaced this node (follow SUPERSEDES edges forward).
   Direction "backward" = what this node replaced (follow SUPERSEDES edges backward).

   Try DuckPGQ:
   ```sql
   -- Forward: follow SUPERSEDES edges from new -> old, starting at this node
   FROM GRAPH_TABLE(memory_graph
       MATCH (a:nodes WHERE a.id = ?)-[e:edges WHERE e.edge_type = 'SUPERSEDES']->+(b:nodes)
       COLUMNS (b.id, b.content, b.lifecycle_state, b.node_type)
   )
   ```
   Note: SUPERSEDES edge direction is new_node -> old_node. So "forward" (what replaced this) means following edges where this node is the TARGET, and "backward" (what this replaced) means following edges where this node is the SOURCE. Adjust query direction accordingly.

   Fallback: Iterative SQL query following superseded_by pointers on nodes table.

**All traversal methods:**
- Use parameterized queries (not f-strings) for node IDs.
- Wrap DuckPGQ calls in try/except, fall back to SQL transparently.
- Log which path (DuckPGQ vs SQL fallback) was used via structlog.

**src/prme/storage/engine.py:**

```python
class MemoryEngine:
    """Unified memory interface. Single entry point for all storage operations."""
```

**Factory method:**
```python
@classmethod
async def create(cls, config: PRMEConfig | None = None) -> "MemoryEngine":
    """Create and initialize a MemoryEngine with all backends."""
```
- If config is None, use PRMEConfig() (loads from env/defaults).
- Open DuckDB connection to config.db_path.
- Call initialize_database(conn) from schema.py.
- Create EventStore(conn).
- Create DuckPGQGraphStore(conn).
- Create FastEmbedProvider(model_name=config.embedding.model_name).
- Create VectorIndex(conn, config.vector_path, embedding_provider).
- Create LexicalIndex(config.lexical_path).
- Return initialized MemoryEngine instance.

**Core methods:**

1. **async store(self, content: str, *, user_id: str, session_id: str | None = None, role: str = "user", node_type: NodeType = NodeType.NOTE, scope: Scope = Scope.PERSONAL, metadata: dict | None = None, confidence: float = 0.5) -> str**:

   Auto-propagation pipeline (per user decision):
   a. Create Event model, append to EventStore (MUST succeed -- source of truth).
   b. Create MemoryNode with evidence_refs=[event.id], create in GraphStore.
   c. Index into VectorIndex (node_id, content, user_id) and LexicalIndex (node_id, content, user_id, node_type) in parallel using asyncio.gather.
   d. Return str(event.id).

   If vector/lexical indexing fails, log the error but don't fail the store() call -- event is already persisted. The derived indexes can be rebuilt from the event log.

2. **async search(self, query: str, user_id: str, *, k: int = 10) -> dict**:
   Run vector search and lexical search in parallel (asyncio.gather).
   Return {"vector_results": [...], "lexical_results": [...]} -- Phase 3 builds the hybrid re-ranker.

3. **async get_node(self, node_id: str, *, include_superseded: bool = False) -> MemoryNode | None**: Delegate to GraphStore.

4. **async query_nodes(self, **kwargs) -> list[MemoryNode]**: Delegate to GraphStore.

5. **async get_event(self, event_id: str) -> Event | None**: Delegate to EventStore.

6. **async get_events(self, user_id: str, **kwargs) -> list[Event]**: Delegate to EventStore.

7. **Lifecycle transition pass-throughs:**
   - `async promote(self, node_id: str) -> None`: Delegate to GraphStore.
   - `async supersede(self, old_node_id: str, new_node_id: str, *, evidence_id: str | None = None) -> None`: Delegate to GraphStore.
   - `async archive(self, node_id: str) -> None`: Delegate to GraphStore.

8. **async close(self)**: Close/save all backends. Save VectorIndex, close DuckDB connection.
