## Decisions Made
- **DuckDB JOIN for vector scope filtering:** Used JOIN with nodes table instead of adding scope column to vector_metadata (no migration needed, local DuckDB performance acceptable at MVP scale)
- **Tantivy scope field safe degradation:** Pre-migration documents without scope field simply won't match scope-filtered queries (P8 principle, consistent with epistemic backfill approach)
- **ENTITY/PREFERENCE temporal exemption:** These persistent knowledge types are never excluded by temporal filters during candidate generation (prevents "what did Alice do last week" from excluding Alice entity)
- **Multi-scope iterate-and-union:** query_nodes() takes single Scope, so multi-scope is implemented by iterating scopes and deduplicating (avoids GraphStore Protocol change)
- **Explicit temporal override:** Pipeline prioritizes explicit time_from/time_to params from caller over analysis-derived values
