<objective>
Forward scope and temporal filters from the retrieval pipeline through candidate generation to all four backend generators, ensuring candidates from wrong scopes and expired nodes are excluded before merging.

Purpose: Closes NSPC-05 (namespace filters before returning vector search candidates) and the RETR-03 temporal partial gap. Currently generate_candidates() accepts scope but never forwards it, and temporal filters from QueryAnalysis are never used during candidate generation.

Output: All four backend generators (graph, vector, lexical, pinned) enforce scope and temporal filtering at query time. Entry points accept multi-scope lists for flexible queries.
</objective>

**VectorIndex.search() -- add scope filter (src/prme/storage/vector_index.py):**
- Add `scope: list[str] | None = None` and `time_from: datetime | None = None`, `time_to: datetime | None = None` parameters to `search()` and `search_by_vector()`.
- When scope is provided, change the DuckDB query that fetches `vector_key` from `vector_metadata` to JOIN with the `nodes` table on `node_id` and add `WHERE n.scope IN (?, ?, ...)` with parameterized values. Use research Pattern 2 (JOIN approach, no migration needed).
- When time_from/time_to are provided, add temporal WHERE clauses on the nodes table: `n.valid_from <= ? AND (n.valid_to IS NULL OR n.valid_to > ?)`. For temporal filtering, exempt ENTITY and PREFERENCE node types: add `OR n.node_type IN ('ENTITY', 'PREFERENCE')` to the temporal clause so these persistent types are never excluded by temporal filters (research discretion recommendation).
- Ensure the parameterized SQL uses DuckDB's `IN` syntax correctly for multi-scope lists.

**LexicalIndex.search() -- add scope filter (src/prme/storage/lexical_index.py):**
- Add `scope: list[str] | None = None` parameter to `search()`.
- When scope is provided, extend the tantivy query string from `"query AND user_id:value"` to `"query AND user_id:value AND (scope:val1 OR scope:val2)"`.
- Add `scope` as a TEXT field to the tantivy schema builder in `__init__()` (alongside existing `user_id` and `node_type` fields).
- Update `add_document()` to accept and index `scope: str | None = None`. When scope is provided, add it to the tantivy document. Per research Pitfall 3, existing documents without scope field will simply not match scope-filtered queries (safe degradation per P8).
- Note: Temporal filtering for lexical backend uses post-filter after tantivy search (tantivy has no temporal field). Add `time_from`/`time_to` parameters to `search()` and post-filter results by checking resolved node temporal validity after the tantivy query. This requires returning node_id from tantivy results (already done) and checking temporal validity via the resolved_nodes map in candidates.py (handled in generate_candidates).

- Forward `scope` (as list of string values: `[s.value for s in scope]` when not None) and temporal params to each `_generate_*` function.
