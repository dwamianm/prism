## Decisions Made
- **Cheapest backends for hints:** Cross-scope hints run only vector+lexical (not full 4-backend pass) with reduced k (2x cross_scope_top_n) per research Pattern 3 for performance
- **Graceful degradation:** Cross-scope hint generation wrapped in try/except -- failures logged as warning, never propagate to caller
- **Backend-level test assertions:** Integration tests verify filtering at the DuckDB JOIN level (vector search) rather than relying on lexical search which has known tantivy multi-word query parsing behavior
