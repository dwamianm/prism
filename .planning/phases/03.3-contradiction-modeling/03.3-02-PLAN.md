               # Default: supersedence (temporal_intent="update" or None)
               logger.info(
                   "Supersedence detected: contradicting facts",

**pipeline.py — Annotate CONTESTED candidates with conflict metadata:**

10. After scoring (Stage 5) and before context packing (Stage 6), add conflict metadata annotation for CONTESTED candidates. Import `EdgeType` and `LifecycleState` from `prme.types` and `UUID` from `uuid`:
    ```python
    # --- Stage 5.5: Conflict Metadata Annotation ---
    # Batch-annotate CONTESTED candidates with conflict_flag and contradicts_id
    contested_ids = [
        str(c.node.id) for c in scored
        if c.node.lifecycle_state == LifecycleState.CONTESTED
    ]
    if contested_ids:
        for cid in contested_ids:
            # Look up CONTRADICTS edges in both directions
            edges_out = await self._graph_store.get_edges(
                source_id=cid, edge_type=EdgeType.CONTRADICTS
            )
            edges_in = await self._graph_store.get_edges(
                target_id=cid, edge_type=EdgeType.CONTRADICTS
            )
            all_edges = edges_out + edges_in
            if all_edges:
                # Find the counterpart node ID
                edge = all_edges[0]
                counterpart_id = (
                    str(edge.target_id) if str(edge.source_id) == cid
                    else str(edge.source_id)
                )
                # Annotate the candidate
                for c in scored:
                    if str(c.node.id) == cid:
                        c.conflict_flag = True
                        c.contradicts_id = UUID(counterpart_id)
    ```

    Per locked decision: "Contradicting counterparts are NOT automatically pulled into results — only included if independently relevant to the query." This code only annotates existing candidates; it does NOT fetch or inject the counterpart into results.

11. Per research discretion recommendation #4 (No scoring penalty): Do NOT modify the scoring formula. CONTESTED objects score normally. The conflict_flag metadata is purely informational for the consuming LLM.
