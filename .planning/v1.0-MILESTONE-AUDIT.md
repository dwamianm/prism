## Cross-Phase Integration

### GAP-01 (Critical): Config values disconnected from retrieval

**Affected:** TRST-07, EPIS-05
**Path:** `PRMEConfig.epistemic_weights` / `PRMEConfig.unverified_confidence_threshold` → stored on `MemoryEngine` → **NOT** forwarded to `RetrievalPipeline`

`RetrievalPipeline.__init__` accepts `scoring_weights` and `packing_config` but not `epistemic_weights` or `unverified_threshold`. The `retrieve()` method calls `filter_epistemic()` and `score_and_rank()` without these parameters — module-level constants are always used regardless of config.

**Impact:** Env vars `PRME_EPISTEMIC_WEIGHTS` and `PRME_UNVERIFIED_CONFIDENCE_THRESHOLD` have zero runtime effect on retrieval. 3/5 config fields work (scoring, packing, confidence_overrides); 2/5 are disconnected.

**Fix:** Add `epistemic_weights` and `unverified_threshold` to `RetrievalPipeline.__init__`, forward in `retrieve()`, wire in `engine.py create()`.

### GAP-02 (High): temporal_intent not forwarded to supersedence detector

**Affected:** EPIS-04
**Path:** `ExtractedFact.temporal_intent` → `pipeline._materialize()` → `detect_and_supersede()` called **WITHOUT** `temporal_intent`

`detect_and_supersede()` is called without `temporal_intent=fact.temporal_intent`. Defaults to `None`, which means the "assertion" branch (creating `CONTRADICTS` edges) is never reached through automated ingestion. Every detected conflict produces a `SUPERSEDES` edge even when the LLM classified it as `"assertion"`.

**Impact:** E2E Flow 3 (Contradiction Detection) is broken for automated ingestion. Direct `graph_store.contradict()` API works. Integration tests pass because they call `contradict()` directly, bypassing the pipeline.

**Fix:** Add `temporal_intent=fact.temporal_intent` to `detect_and_supersede()` call in `pipeline.py`.

### GAP-03 (Medium): Scope not written to lexical index at ingestion

**Affected:** NSPC-05
**Path:** `lexical_index.index()` called **WITHOUT** `scope` in `pipeline.py` and `engine.py`

The `scope` parameter was added to `LexicalIndex.index()` in Phase 03.2, and tantivy scope filtering was implemented at query time. However, no write call site passes scope, so documents lack scope metadata. Scope-filtered lexical queries return no results.

**Impact:** Scope isolation works for graph and vector backends but silently fails for lexical. This was acknowledged during Phase 03.2 as a design trade-off.

**Fix:** Add `scope` parameter to all `lexical_index.index()` calls in `pipeline.py` and `engine.py`.
